TaskFlow - Complete Project Context
Project Summary

TaskFlow is a portfolio-quality full-stack web application built using the MERN Stack (MongoDB, Express.js, React, Node.js).

The purpose of this project is not to become a production-ready project management platform, but rather to demonstrate a solid understanding of modern full-stack web development concepts to potential employers.

The application should showcase:

MERN architecture
REST API development
Authentication & Authorization
MongoDB database modeling
CRUD operations
React component architecture
State management
Form validation
Responsive UI
Modern coding practices
Clean project organization

The application should be simple enough to complete quickly while still appearing polished and professionally structured.

Primary Goal

This project is intended to be used as a demonstration during interviews for entry-level MERN Stack or Full Stack Developer positions.

The emphasis should be on:

Clean architecture
Readable code
Proper folder organization
Industry-standard practices
Maintainability
Scalability

The goal is not to implement every possible feature.

Expected Outcome

The finished application should look like a professional portfolio project that demonstrates competency in MERN stack development. It should be easy to explain during interviews, with clear separation of frontend and backend responsibilities, proper RESTful API design, secure authentication, and clean, maintainable code. The emphasis is on quality over quantity—implement the MVP completely rather than partially implementing advanced features.

Additional Instructions for the MCP Agent
Generate production-quality, readable code with meaningful naming conventions.
Keep components small and reusable.
Use consistent formatting and folder organization.
Include comments only where they improve understanding; avoid excessive commenting.
Prefer reusable custom hooks and utility functions over duplicated logic.
Build incrementally so the application remains runnable after each major feature.
Follow REST conventions and return consistent JSON responses.
Assume this is a portfolio project intended to showcase good engineering practices, not an enterprise application.
